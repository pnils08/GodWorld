# /sift v2 Rebuild Plan — Reporter-Agency + Sheet-Primary + Civic-Opt-In

**Goal:** Replace /sift v1.2 with v2 — a brief generator built for the reporter-agency model that reads sheet-primary inputs, treats civic content as one slice (not the spine), emits canonical brief shape with newspaper-section ordering + cadence caps + dispatch.json + Memory-Fence wrap, and triages by fold / covered-by-feature vocabulary rather than slate variants.

**Architecture:** /sift v1.2 was built for a "city-hall paper" load-out — civic content over-weighted, world_summary as primary input, brief_template assumes desk-curated citizen pre-rolls. S220–S222 surfaced the model mismatch: 5-variant slate loops, JSON pre-approval ambiguity, multi-article-per-reporter briefs, letters-desk candidate-pool collisions with rest-cycle rule, no dispatch.json emission, untitled proposals breaking djDirect title-matching. v2 inverts: sheet primary, world_summary orientation-only, civic as one of six sections, reporter-agency briefs (signal + voice direction), cadence cap baked into slate selection, mandatory headline + section + reporter assignment per proposal, dispatch.json + Memory-Fence wrap emitted as deterministic outputs of Step N.

**Terminal:** research-build (skill text + template rewrite, multi-session). Upstream sheet-primary + canon archive + NEWSROOM_MEMORY load-bearing — work depends on `lib/sheets.getSheetData()` already in place (verified S221).

**Pointers:**
- Prior work: `.claude/skills/sift/SKILL.md` v1.2 (current state)
- Brief template: `docs/media/brief_template.md` (pre-reporter-agency, drift surface)
- Triage master: [[2026-05-22-c94-gap-log-triage]] §3 C2
- ADR-0006: [[../adr/0006-parser-validator-format-contracts]] — every brief-format contract either ships canonical exemplar OR fails loud
- Related cluster: C3 (world_summary fidelity) — sift Step 1 sheet-primary discipline pairs with engine-side world_summary cleanup
- Related cluster: C4 (format-contract docs) — sift's dispatch.json + brief contract land together with /write-edition's compile template
- Related cluster: C7 (canon drift) — sift Step 4 enrichment embeds employer biz in citizen brief to close G-W35 fabrication surface
- S215 brief-led mode (validated G-W41) — don't roll back; v2 keeps brief-led, fixes upstream brief production

**Acceptance criteria:**
1. /sift v2 SKILL.md reads sheet primary (`Oakland_Sports_Feed`, `Riley_Digest`, `Simulation_Ledger` + initiative tracker) as Step 1, with world_summary downgraded to orientation-only.
2. Brief shape canonicalized: one brief per article (no multi-article-per-reporter files), each carries headline + newspaper-section + reporter assignment + 1-paragraph signal + voice direction + Memory-Fence wrap.
3. dispatch.json emitted by /sift Step N with reporter→article→file mapping; /write-edition reads it directly (no fallback title-matching).
4. Cadence caps baked in: per-reporter max + per-newspaper-section cap; civic content capped at 2-3 articles (one slice, not spine).
5. Letters-desk receives candidate-pool brief (canonical shape: letters-candidates.md), with explicit rest-cycle awareness — letters-desk LENS still owns final selection (S215 self-defense pattern preserved).
6. Triage vocabulary: `fold` (combine two proposals into one), `covered-by-feature` (drop because section feature absorbs it), `defer-to-supplemental` (interview/dispatch handles it) — eliminates 5-variant slate loops by giving structured reduction primitives.
7. Brief omits prose-template body; replaces with bullet-form signal + 3-5 voice-direction lines (no skeletonized article).
