## Overview

GodWorld is a city-simulation engine (v2.10.0) that models a fictional Oakland (primary) and Chicago (satellite) across "cycles" (in-world time periods). Each cycle runs a 11-phase JavaScript pipeline that advances demographics, citizen lives, civic events, and media—then writes results to Google Sheets and publishes an in-world newspaper called *The Cycle Pulse*. The codebase blends Google Apps Script (GAS via clasp), a local Node.js runner, a Vite dashboard, and heavy AI-model integration (Claude, Gemini, OpenRouter).

## Architecture Notes

### Simulation Loop
Each "cycle" is one in-world time period (~2 weeks fictional). The loop is:
1. **Boot** (P1): load config, advance calendar, init ledger
2. **World State** (P2): update weather, sports, economy
3. **Population** (P3): demographic drift, crime, city tier
4. **Events** (P4): generate city/citizen/faith/generational events
5. **Citizens** (P5): simulate named citizens — bonds, career, household, elections
6. **Analysis** (P6): filter noise, detect patterns, economic ripple, arc lifecycle
7. **Evening Media** (P7): generate story seeds → media room briefing → Cycle Pulse edition
8. **Chicago** (P8): run satellite city, apply cooldowns and recovery
9. **Digest** (P9): compress and weight the cycle
10. **Persistence** (P10): export artifacts, compile handoff, commit ledger
11. **Media Intake** (P11): parse incoming media notes, health cause intake

### Two Environments
- **Oakland** (primary): Full simulation, ~66 sheets
- **Chicago** (V3 satellite): Lighter simulation, dependent on Oakland state

### AI Integration Pattern
- Phases 5 and 7 make heavy Anthropic API calls (citizen dialogue, story generation)
- `lib/mags.js` is the Supermemory interface — stores per-citizen "magazines" (memory containers)
- `scripts/godworld-mcp.py` exposes a Python MCP server so Claude Code can call GodWorld tools directly

### Test Architecture
Tests use a custom runner (`scripts/run-tests.js`). Files follow naming conventions:
- `*.test.js` — unit tests
- `*.contract.test.js` — contract/integration tests
- `*.gate.test.js` — gate tests (must pass before deploy)

### Multi-Terminal AI Workflow
The `.claude/terminals/` directory manages multiple concurrent Claude sessions ("terminals"), each working a different domain (engine-sheet, research-build, etc.). Session state is tracked in `.claude/state/`, skills in `.claude/skills/`.
