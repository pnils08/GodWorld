## §4 — Reads-without-writes (silent-undefined bugs)

**Status: DEFERRED — current regex is too loose to ship findings.**

The skill's `S.\w+` and `ctx.summary.\w+` patterns over-match. Many phase files alias `S` to other objects (constants packs, state machines, lookup tables — `S.ACTIVE`, `S.HS_DIPLOMA`, `S.FESTIVAL`, etc.), and the regex also picks up method calls (`.length`, `.push`, `.indexOf`, `.join`, `.hasOwnProperty`) on summary-field accessors. The unfiltered output is 139 candidate "unwritten reads," dominated by these false positives.

**Action item:** before the next audit, refine the §4 regex to (a) scope to files where `const S = ctx.summary` (or equivalent) is established, (b) exclude method-call suffixes, (c) cross-reference reads against destructuring-style writes (`const { foo } = ctx.summary`). Track as a follow-up to the skill itself rather than an engine bug.
