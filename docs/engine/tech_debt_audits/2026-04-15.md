## INFO — refactoring opportunities

- `Math.random()` fallback pattern (warn+call) appears in 3 files with near-identical shape (`generationalEventsEngine.js`, `generateGenericCitizens.js`, `v2DeprecationGuide.js`). Candidate for a shared `safeRand_(ctx)` helper in `utilities/` that enforces ctx.rng presence or throws.
- `ctx.summary = S;` assignment pattern at function bottoms (20+ occurrences) is ceremonial — `S` is already a reference to `ctx.summary`, so reassigning to itself is a no-op. Not a bug, but reads as defensive code whose purpose has faded.
