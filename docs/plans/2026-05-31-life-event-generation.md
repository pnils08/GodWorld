# Life-event generation ‚ÄĒ audit + the traits‚Üíevents back-arc

## Context

The citizen-essence loop has three sides. engine.31 is the **compressor** (O‚ÜíR). engine.30 is the **earned story** (R‚ÜíAT). This plan is the **event-generation side** ‚ÄĒ the **O producers** ‚ÄĒ and, crucially, the **missing back-arc that makes the loop bidirectional:** today events shape traits (O‚ÜíR); Mike's S249 foresight is that **traits should shape events** (R‚Üíevents), so a citizen's personality begets characteristic events and **the events make sense to that citizen's life.**

**Headline finding (verified S249):** `compressLifeHistory.js` ships `getCitizenArchetype_`, an accessor commented *"for event generation"*. It **is** called ‚ÄĒ but only by `phase07-evening-media/storyHook.js` (3 sites). **No event *generator* (`phase04-events/*`, `phase05-citizens/*`) calls it.** So traits already shape *story hooks* ‚ÄĒ the pattern works and is proven ‚ÄĒ but the **traits‚Üíevent-generation arc specifically is unwired.** (Two generators, `generateGameModeMicroEvents` + `generateCitizensEvents`, read `TraitProfile` ad-hoc, not via the accessor.) Mike's #5 is the real gap; the working accessor makes it cheap to wire.

Closing this turns the loop into a true cycle: **events ‚Üí R ‚Üí AT, and R ‚Üí events.** A deployed citizen then both *reads* as itself (R+AT) and *keeps living* in-character (R-shaped events).

## Scope

**In:** an audit of the life-event system across all generators (what fires, and how clock / fame / traits / circumstances gate it), a slot-coverage check against the 8-slot memory model, and the design to wire **traits‚Üíevents** with stability damping.

**Out:** the compressor tag fixes (engine.31), the R‚ÜíAT step (engine.30). This is the generator side. All execution is engine-sheet, gated.

## The foreseen items (T1‚ÄďT6, S249) + the S250 Conduct addition (T7)

- **T1 ‚ÄĒ Audit the life events all engines output (NEW).** No inventory exists. Catalogue every event each generator emits (`phase04-events/*`, `phase05-citizens/*`) ‚ÄĒ the event, its trigger, its tag, its target. The tag audit (engine.31) covered *tags*; this covers the *events* and their selection logic. Output: a single event-system map. **Mike S249:** the inventory should also map, **per Simulation_Ledger column**, what random outcomes could fire off it ‚ÄĒ every column is a potential event axis (the chaos-cars per-column generalization), so the audit doubles as a column‚Üíoutcome-space map for the random-event engine.
- **T2 ‚ÄĒ Clock factor (WELL-WIRED ‚Üí audit for coherence).** `ClockMode` is read across ~12 generators (`generateGenericCitizenMicroEvent`, `generateMediaModeEvents`, `generateCitizensEvents`, `generationalEventsEngine`, `runEducationEngine`, `runNeighborhoodEngine`, `runRelationshipEngine`, `runHouseholdEngine`, `runCareerEngine`, `checkForPromotions`, ‚Ä¶) and the mode-specific generators (Media/Game/Civic) *are* the clock split. So clock is the most-wired factor ‚ÄĒ the work is auditing **consistency/coherence**, not building it. Confirm ENGINE-mode citizens (the life-loop's audience) have a full event vocabulary.
- **T3 ‚ÄĒ Fame factor (GAP ‚Üí likely build).** `UsageCount`/fame is read by **intake/context** code (`processIntakeV3`, `citizenContextBuilder`, `processAdvancementIntake`) ‚ÄĒ **not by the event generators.** So fame shapes intake/context but **does not clearly gate event selection** today. Audit confirms, then likely **build**: famous citizens should draw different events than invisible ones; wire it coherently with the fame system (engine.29).
- **T4 ‚ÄĒ Memory-slot coverage (NEW additions).** Map events ‚Üí the **5 memory categories (Social / Work / Family / Health / Conduct** ‚ÄĒ the 5th added in the engine.31 S250 redesign). engine.31 found **Work has no live tag**, **Family/Health are thin**, and **Conduct has ZERO output** (no generator emits a crime ‚ÄĒ confirmed S250). **If a slot has no output, add the generation for it** (Mike's directive). The Conduct gap is the largest and is broken out as its own marquee task ‚Üí **T7**. Output: a coverage matrix + new generators/events for the gaps.
- **T5 ‚ÄĒ Traits‚Üíevents back-arc (NEW ‚ÄĒ the big one).** Wire R (archetype + trait axes) into event *selection*, so personality shapes which events a citizen draws. The accessor exists and **already works** (`getCitizenArchetype_`, used by `storyHook.js`) ‚ÄĒ it's just **never called by an event generator**; two generators (`generateGameModeMicroEvents`, `generateCitizensEvents`) read `TraitProfile` ad-hoc. So this is wiring a proven accessor into the generators, not building from scratch. **‚ö† Stability guard:** this makes the loop a feedback system (traits‚Üíevents‚Üítraits) ‚ÄĒ it can amplify into runaway or cookie-cutter sameness. Borrow the chaos-cars **asymmetric-decay / anti-cookie-cutter** discipline (engine.11) and the bounded-memory **promotion ladder** (engine.31 Phase 3 ‚ÄĒ only sustained *core* memories compound; non-core decays) to damp it. **Flagship case of the back-arc = the Conduct momentum loop (T7):** R's conduct-core biases the next moral test's commit-vs-resist, severity, and frequency (the dark-momentum, "bad begets bad"). The Emergence/Grok drift lesson applies: tighter feedback over long horizons makes the canon/bounding layer more load-bearing.
- **T6 ‚ÄĒ Event coherence (NEW).** Events must make sense for the citizen ‚ÄĒ age, job, family, neighborhood, *and* traits (T5). A retired 68-year-old shouldn't draw a youth event; an Anchor-archetype homebody shouldn't draw constant nightlife. Depends on T5 + the generators reading circumstance fields (partially done). Output: coherence gates in event selection.
- **T7 ‚ÄĒ The Conduct generator: the moral-test engine (NEW ‚ÄĒ S250, the marquee build).** Conduct (the 5th memory category) has zero event output today ‚ÄĒ the city carries a `Crime_Metrics` *rate* but not one tracked citizen ever commits a crime, so every one of 903 lives is, mechanically, clean. This builds the **low-probability moral-test generator**: at a slow base rate a citizen draws a moral test that resolves ‚ÄĒ **R-biased (T5)** ‚ÄĒ as either a **transgression** (severity ladder: `Transgression-Petty` stole-butter ‚Üí `-Serious` theft/fraud ‚Üí `-Grave` embezzlement/violence) or a **`Resisted`** (tempted, didn't ‚Üí accretes integrity). Emits against the `TAG_REGISTRY` Conduct vocab; engine.31's bounded-memory engine receives it, and a *sustained pattern* promotes to a **dark core** (`Outlaw`) ‚ÄĒ "no one is safe" becomes mechanism here, while the bound keeps a single petty act from damning anyone. **Population counterweight (RimWorld adaptation-factor, `research4_1.md:211`):** the per-citizen bound stops *individual* runaway, but city-level "bad begets bad" has no floor ‚ÄĒ when `Crime_Metrics` spikes, the engine should lean positive (opportunities / interventions / community wins), a real city's resilience. Without it the moral-momentum can cascade the whole tracked cohort dark. **Depends on engine.31's Conduct slots existing (engine.31 Phase 4); builds after.** This is the dark thread the whole arc was for.

## Acceptance criteria
1. A complete life-event inventory exists (T1) ‚ÄĒ every generator's events, triggers, tags, targets, and clock/fame/trait/circumstance gating in one map.
2. Every memory-slot category has event output, or a documented decision not to (T4).
3. Clock + fame gating is audited and gap-closed (T2/T3).
4. `getCitizenArchetype_` (or its successor) is called **from event generators** (not only storyHook) ‚ÄĒ traits demonstrably shape event selection (T5).
5. **Loop-stability is MEASURED, not asserted (the gate with teeth).** Damping "in place" is unfalsifiable ‚ÄĒ instead, once the back-arc is live, **run N citizens for M cycles with R‚Üíevents active and measure whether trait distributions converge, diverge, or oscillate** (same discipline as the research.13 PoC: don't certify the loop closed until you've watched it run closed). The Emergence/Grok lesson is that nobody can predict which way three coupled feedback paths (events‚ÜíR‚Üíevents, R‚ÜíAT) run until they run. T5 does not ship until this measured run passes.
6. Event-coherence gates reject circumstance-mismatched events (T6) ‚ÄĒ spot-check: no youth events on retirees, no nightlife spam on homebodies.
7. **The Conduct generator produces crime (T7).** Moral tests fire at a low base rate, resolve R-biased to transgression (with severity) or resistance; a *sustained* transgression pattern promotes to a dark core while a single petty act does not; and the city-level counterweight measurably leans positive when `Crime_Metrics` spikes (no runaway cohort-wide darkening). Same measured-run discipline as #5 ‚ÄĒ verify on a multi-cycle trace, don't assert.
